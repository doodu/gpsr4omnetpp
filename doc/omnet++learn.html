1  如何编译?(Windows平台下)

1.生成Makefile: 

cd c:/blueq/bin/OMNeT++ 
cd ./tutorial/queues 
opp_nmakemake -f 

2.编译 

nmake 

3.运行 

queues.exe 

程序启动后会有两个窗口: 一个图形窗口,用于显示网络 另一个窗口名叫OMNeT++/TKenv,是主窗口 4.分析模拟结果 两个文件保存了结果:

USERIF\_LIBS=\$(CMDENV\_LIBS) 
#USERIF\_LIBS=\$(TKENV\_LIBS)

make clean && make 

你要在omnetpp.ini中设置下面的变量: 

[Cmdenv] 
runs-to-execute = 1 
mofule-messages = no

3  M/M/1模拟器的实现

理解代码的一个简单的方法:

  1. 运行,理解它是如何运作的

  2. 看.ned文件

  3. 看.h文件

  4. 看.cc文件

  1. 功能实体(functional entiy):模拟器的一部分,在任务中xie带一个特定的动作或者消息.

  2. 模块(modules):功能实体的实现.

  3. 模拟的环境主要由两种文件构建,.ned文件和.cc文件:

  4. .ned文件:描述各个功能实体之间如何连接以及如何通信的

  5. .cc文件:包含了实体的实现代码

4  为什么使用OMNeT++

  1. 它的开发语言是c++

  2. 它是开源的

  3. 具有良好的可扩展性

  4. 它使用GPL license

  5. 它强制各个模块相互分离,这正是网络拓扑结构所要求的
